# User Preferences

## Coding Style
- Prefer smaller functions
- Prefer functional programming patterns
- Do not delete existing comments unless you just resolved them

### Test Code
- Ensure that tests produce legible error output, where possible
  - Prefer value expectations or even custom matchers over boolean expectations
  - Use collection based expectations wherever possible.

## Code Review
- Prefer to use built-in functions or functions of already included libraries
- Always check for opportunities to re-use and simplify code
- Less code is better

## Communication
- Be concise in explanations
- Reference standards where appropriate

## Execution
- If you got a request with multiple subtasks, start with the first one and
  don't think too long about the later ones
- If you find yourself revisiting different aspects of a task, stop and
  ask a question
